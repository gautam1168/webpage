# Rules of naming 1. Function names should be verbs and class names should be nouns ```csharp class Graph { private graphConfiguration: Object; private eventQueue: Array; constructor() {} render() {} addEventListener() {} } ``` 2. Each concept should have one name and one name only 3. Say what you mean and mean what you say (don't pun or be cute) 4. Take time to differentiate names when conflicts arise 5. Don't encode 6. Add context but not too much 7. Don't use noise words like abcManager or xyzData in names # Rules for making functions 1. Functions should be small (~20 lines max) 2. Names of functions should be verbs 3. Functions shouldn't take more than 2 arguments # Rules for writing comments 1. Only write comments when you cannot say it in code 2. Don't make javadocs for non public functions # Rules for writing classes 1. They should be small. 2. They should be cohesive.
